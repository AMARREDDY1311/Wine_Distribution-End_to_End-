01 — OverviewProject Summary

Brescome Barton Inc. is the largest wholesale distributor of spirits, wine, and beer in Connecticut (~$348M revenue). This warehouse consolidates sales rep performance, supplier incentive programs, customer accounts, and transaction data into a single analytics-ready platform.

🥉 Bronze Layer

Raw data ingested as-is from 8 CSV source files. All columns are NVARCHAR — no type enforcement. Loaded via stored procedure bronze.load_bronze.

🥈 Silver Layer

Cleansed, typed, deduplicated, and validated. Types enforced, nulls handled, values standardized. Loaded via silver.load_silver.

🥇 Gold Layer

Analytics-ready Star Schema views. 6 dimensions + 3 fact tables. Surrogate keys generated via ROW_NUMBER(). Ready for Power BI / Tableau.

📊 Source Data

8 CSV files — 16,292 total rows. Covers sales reps, suppliers, products, accounts, incentive programs, transactions, quotas, and program attainments.

📅 Date Range

January 2024 through December 2025. Two full years of daily transaction data with seasonal patterns baked in (holiday peaks, summer peaks).

🏗️ Tech Stack

Microsoft SQL Server (SQLEXPRESS01), T-SQL, SSMS. Stored procedures for ETL. Views for Gold layer. No external dependencies.

03 — Data FlowEnd-to-End Pipeline

Data moves from 8 source CSV files through three layers via stored procedures. The full pipeline can be re-run at any time — all loads are idempotent (Truncate + Insert).

📁

Source Files — 8 CSV Files

01_sales_reps.csv · 02_suppliers.csv · 03_products.csv · 04_accounts.csv · 05_supplier_programs.csv · 06_sales_transactions.csv · 07_rep_quota_actuals.csv · 08_program_attainments.csv — stored in C:\BresCome_DW\datasets\

1

EXEC bronze.load_bronze — Raw Ingestion

BULK INSERT reads CSVs into bronze tables. All columns loaded as NVARCHAR. dwh_create_date = GETDATE() stamped on each row. Truncate + Insert strategy ensures clean reload every run.

2

EXEC silver.load_silver — ETL Layer 1

Reads from bronze, applies: TRY_CAST for type safety · LTRIM/RTRIM on all strings · CASE standardization for channel/status values · ROW_NUMBER() deduplication per natural key · COALESCE for null handling · Derived columns (performance_flag, is_valid). Truncate + Insert into silver tables.

3

ddl_gold.sql — Star Schema Views

CREATE OR REPLACE VIEW statements build 6 dimension views and 3 fact views on top of silver. Surrogate keys via ROW_NUMBER(). Business logic (price_tier, origin_type, gap_to_goal, bonus_missed_usd). No physical load — views always read from current silver data.

✓

Power BI / Tableau — Analytics

Connect directly to BresCome_DW · schema: gold. All 9 objects (6 dims + 3 facts) are immediately available. Set relationships in Power BI using surrogate keys (rep_key, account_key, product_key, supplier_key, program_key).

07 — QualityData Quality Checks

Run quality_checks_silver.sql after every EXEC silver.load_silver. All queries should return 0 rows (or pass conditions listed below).

🔍 Section 1 — NULL / Missing Keys

NULL rep_id in silver.sales_reps = 0
NULL txn_date in silver.sales_transactions = 0
NULL / zero revenue_usd in transactions = 0
NULL price_per_case_usd in products = 0
NULL goal_value or bonus_usd in programs = 0

🔍 Section 2 — Duplicate Keys

Duplicate rep_ids in silver.sales_reps = 0
Duplicate txn_ids in transactions = 0
Duplicate sku_ids in products = 0
Duplicate rep+month in quota table = 0

📅 Section 3 — Date Validation

Transactions outside 2024–2025 range = 0
Programs with start_date > end_date = 0
Reps with hire_date in the future = 0

✅ Section 4 — Standardization

Channel values = only "On-Premise" or "Off-Premise"
Supplier categories = Wine / Spirits / Wine & Spirits
performance_flag = Achieved / At Risk / Behind only
Attainment status = same 3 values only

🔗 Section 5 — Referential Integrity

is_valid = 0 transactions logged and reviewed
Orphan transactions (rep not in sales_reps) = 0
Orphan accounts (rep not in sales_reps) = 0
Attainment pct matches actual/goal ratio (±1%)

📊 Section 6 — Row Count Validation

silver.sales_reps = 15
silver.suppliers = 15
silver.products = 25
silver.accounts = 185
silver.sales_transactions = 15,000

08 — Run OrderHow to Run the Pipeline

Run scripts in this exact order. Steps 1–2 are one-time setup. Steps 3–5 are the repeatable pipeline.

1

One-Time: Create Database and Schemas

Run init_database.sql — creates BresCome_DW database with bronze, silver, and gold schemas.

-- Run once
exec init_database.sql
2

One-Time: Create All Tables and Views

Run DDL scripts to create all physical tables and Gold views.

exec scripts/bronze/ddl_bronze.sql
exec scripts/silver/ddl_silver.sql
exec scripts/gold/ddl_gold.sql        -- includes fixed dim_date
3

Load Bronze — Raw Ingestion

Place 8 CSV files in C:\BresCome_DW\datasets\. Grant NT Service\MSSQL$SQLEXPRESS01 Read permission on that folder. Then run:

EXEC bronze.load_bronze;
4

Load Silver — ETL Layer 1

Transforms bronze into typed, cleansed, validated silver tables.

EXEC silver.load_silver;
5

Validate Silver

Run quality checks. All checks should return 0 rows / expected values.

-- Run quality_checks_silver.sql manually in SSMS
6

Query Gold — Analytics Ready

Gold views are always fresh (no load step needed). Connect Power BI to BresCome_DW → schema: gold.

SELECT * FROM gold.fact_sales;
SELECT * FROM gold.dim_reps;
SELECT * FROM gold.fact_quota;
SELECT * FROM gold.fact_attainment;
